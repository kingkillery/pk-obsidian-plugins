# How To Learn With NotebookLM

## Goal

Use NotebookLM to compress the distance between raw source material and durable understanding.

## Workflow

1. Start with a narrow question, not a vague topic.
2. Add only the sources needed for that question.
3. Ask NotebookLM for contrasts, misconceptions, and examples.
4. Convert answers into retrieval prompts, not passive summaries.
5. End each session by writing what you can explain without the sources.

## What good learning looks like

- You can explain the idea in plain language.
- You can compare it against adjacent ideas.
- You can produce examples and counterexamples.
- You can recall it later without reopening the source.

## Anti-patterns

- dumping an entire subject into one notebook
- collecting summaries without questions
- rereading outputs instead of testing recall
- asking for polished prose before building understanding

## Core prompt pattern

> Teach this as if I need to explain it tomorrow.
> Show the big idea, the moving parts, the common confusion, and a short retrieval drill.
